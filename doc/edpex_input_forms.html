Module Explanation: EdPEx Data Entry System

Concept: ระบบนี้ถูกออกแบบมาเพื่อรองรับการกรอกข้อมูลตามเกณฑ์ EdPEx ที่มีความหลากหลายของข้อมูล โดยแบ่ง Form ออกเป็น 5 Block หลัก (A-E) เพื่อให้ครอบคลุมทุกมิติของตัวชี้วัด

  • Block A (Standard Numeric): ใช้สำหรับเก็บข้อมูลตัวเลขทั่วไปที่เป็น Time-Series เช่น ร้อยละผลสอบ, จำนวนผู้รับบริการ
  • Block B (Matrix/Dimensional): ใช้สำหรับข้อมูลที่มีการจำแนกกลุ่มย่อย เช่น จำนวนบัณฑิตแยกตามสาขา, ความพึงพอใจแยกตามกลุ่มลูกค้า
  • Block C (Milestone Project) [NEW]: ออกแบบใหม่เพื่อรองรับ KPI เชิงโครงการที่มีความซับซ้อน เช่น โครงการวัคซีน (7.4.13) ที่ต้องติดตาม Phase, Status, และ % Progress
  • Block D (Narrative): ใช้สำหรับข้อมูลเชิงบรรยาย เปรียบเทียบ Before/After เช่น การปรับปรุงหลักสูตร
  • Block E (Financial & Strategy) [NEW]: เพิ่มเข้ามาเพื่อรองรับหมวด 7.5 ที่ขาดหายไป เน้นข้อมูลการเงินและ Ranking

Data Processing: เมื่อผู้ใช้กรอกข้อมูล ระบบจะแปลงเป็น JSON Object และบันทึกลงใน Collection kpi_entries โดยมี field type เป็นตัวระบุโครงสร้างข้อมูล (numeric, matrix, milestone, narrative, financial) เพื่อให้สามารถ Query ไปแสดงผลบน Dashboard ได้อย่างถูกต้อง
