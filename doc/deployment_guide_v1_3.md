# ขั้นตอนการตรวจสอบและส่งงานขึ้น GitHub (v1.3)

เพื่อให้มั่นใจว่าโปรเจกต์พร้อมสำหรับการใช้งานและเก็บสำรองข้อมูลไว้บน GitHub อย่างถูกต้อง รบกวนคุณดำเนินการตามขั้นตอนดังนี้ครับ:

### ขั้นตอนที่ 1: ตรวจสอบการ Build (Build Verification)
รันคำสั่งนี้เพื่อเช็คว่าโค้ดทั้งหมดในเวอร์ชัน 1.3 ไม่มี Error และพร้อมสำหรับการ Deploy:
```bash
npm run build
```

---

### ขั้นตอนที่ 2: บันทึกและส่งงานขึ้น GitHub (Git Push)
รันคำสั่งตามลำดับด้านล่างเพื่อ Commit งานทั้งหมดที่ทำในวันนี้ (Scopus API + A4 Export + V1.3 Docs):

1. **เตรียมไฟล์ทั้งหมด**:
   ```bash
   git add .
   ```

2. **บันทึกงาน (Commit)**:
   ```bash
   git commit -m "feat: complete research module phase 1 (v1.3.0) - scopus sync & a4 export"
   ```

3. **ส่งขึ้น GitHub**:
   ```bash
   git push origin main
   ```

---

### รายการสิ่งที่จะถูก Push ขึ้นไป:
- **Source Code**: ระบบสืบค้น Scopus, ระบบ Export A4, การอัปเดตเวอร์ชัน v1.3 ทั่วทั้งระบบ
- **Documents**: คู่มือการใช้งานตัวใหม่ (`user_research_manual.md`) และบันทึกเวอร์ชัน (`version_log.md`)
- **Metadata**: รายการงานที่ทำเสร็จสิ้นในวันนี้

*ยินดีที่ได้ร่วมงานในเวอร์ชันนี้ครับ! หากคุณรันคำสั่งเหล่านี้เสร็จแล้ว งานทั้งหมดจะถูกสำรองไว้บน Cloud อย่างปลอดภัยครับ*
