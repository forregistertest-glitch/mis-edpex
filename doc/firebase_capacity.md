## สรุป

ระบบ KUVMIS สามารถใช้ทดสอบจริงกับเจ้าหน้าที่ 5-10 คน เป็นเวลา 1-2 เดือน
โดยไม่มีค่าใช้จ่าย (Firebase Spark Plan = ฟรี, ไม่ต้องใส่บัตรเครดิต)

---

## Quota ฟรีของ Firestore (Spark Plan)

| รายการ             | ฟรีต่อวัน   | ใช้จริง (5-10 คน) | เหลือ          |
|--------------------|:-----------:|:------------------:|:--------------:|
| อ่าน (reads)       | 50,000      | ~500-2,000         | เหลือเยอะมาก  |
| เขียน (writes)     | 20,000      | ~50-200            | เหลือเยอะมาก  |
| ลบ (deletes)       | 20,000      | ~0-10              | ไม่กระทบ       |
| พื้นที่เก็บ         | 1 GB        | ~5-10 MB           | < 1%           |
| Bandwidth          | 10 GB/เดือน | ~100 MB            | < 1%           |

---

## การประมาณการใช้งาน

- 5-10 คน กรอกข้อมูลวันละ 10-20 รายการ → ใช้ไม่ถึง 1% ของ quota ฟรี
- ทดสอบ 1-2 เดือน → ไม่มีค่าใช้จ่ายเลย
- ไม่ต้องใส่บัตรเครดิต จนกว่าจะอัปเกรดเป็น Blaze Plan

---

## สิ่งที่ควรรู้สำหรับ Pilot Test

### 1. Firestore Rules
ตอนนี้ Firestore Rules ยังเป็น open access (ใครก็เขียนได้ถ้ามี URL)
สำหรับ 1-2 เดือนแรกไม่เป็นปัญหา เพราะเป็นการทดสอบภายใน
แต่ถ้าจะเปิดใช้จริง ควรเพิ่ม Security Rules

### 2. ระบบ Login (ดำเนินการแล้ว ✅)
ระบบมี Firebase Authentication + Google Sign-In + Email Whitelist เรียบร้อยแล้ว
รวมถึง Role-based Access Control (user / reviewer / admin) และ Login Logs พร้อม IP Geolocation

### 3. ข้อมูลจริง
ข้อมูลที่กรอกเข้าไปจะถูกเก็บบน Firestore จริงทั้งหมด
- ไม่หาย แม้ปิดหน้าเว็บ
- ดึงออกมา export เป็น Excel ได้ตลอดผ่านหน้า Reports
- มี audit trail (ใครกรอก, เมื่อไหร่, ค่าอะไร)

### 4. Geolocation API (ip-api.com)
ระบบใช้ ip-api.com (Free Tier) สำหรับดึงตำแหน่ง IP → จำกัด 45 requests/นาที
เพียงพอสำหรับ 5-10 คนเข้าสู่ระบบ ไม่มีค่าใช้จ่าย
