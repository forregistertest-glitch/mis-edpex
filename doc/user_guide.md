## 1. การเริ่มต้นใช้งาน (Getting Started)
### 1.1 การเข้าสู่ระบบ (Login)
- เปิดเว็บไซต์ KUVMIS และกดปุ่ม **"Sign in with Google"**
- เลือก Google Account ของมหาวิทยาลัย (@ku.th) ที่ได้รับอนุญาต
- ระบบจะตรวจสอบสิทธิ์และพาเข้าสู่ Dashboard ตาม Role ของท่าน (User, Reviewer, Admin)

### 1.2 แถบเมนูด้านข้าง (Sidebar)
- **Dashboard**: สรุปภาพรวม KPI (Executive View)
- **Academic / Staff / Hospital / Strategic**: เจาะลึก KPI แต่ละหมวดแบบ Interactive
- **กรอกข้อมูล**: ฟอร์มสำหรับป้อนค่าตัวชี้วัดเข้าสู่ระบบ
- **ตรวจสอบข้อมูล**: เมนูสำหรับผู้อนุมัติ (Reviewer ขึ้นไป)
- **จัดการผู้ใช้ / ระบบบุคลากร / ระบบนิสิต**: โมดูลบริหารจัดการ Master Data

---

## 2. การจัดการข้อมูลสารสนเทศ (KPI & Data Management)
### 2.1 การกรอกข้อมูล (KPI Input)
- เลือกฟอร์มที่ต้องการจาก Grid (7 รูปแบบตามประเภทข้อมูล)
- กรอกตัวเลขหรือข้อมูลตามช่องที่ปรากฏ (แนะนำให้กรอกให้ครบถ้วนเพื่อความแม่นยำของกราฟ)
- **สถานะข้อมูล**: 
    - 🟡 **Pending**: รอการตรวจสอบจาก Reviewer
    - 🟢 **Approved**: อนุมัติแล้ว (จะไปปรากฏที่ Dashboard ทันที)
    - 🔴 **Rejected**: ถูกปฏิเสธ (สามารถดูเหตุผลและแก้ไขส่งใหม่ได้)

### 2.2 การตรวจสอบและอนุมัติ (Approval Workflow)
- **Reviewer** สามารถเลือก อนุมัติ (Approve), ปฏิเสธ (Reject) หรือส่งกลับแก้ไข (Revision)
- ข้อมูลที่ถูกตัดสินใจแล้วจะมีการบันทึก Audit Trail (ใคร อนุมัติ เมื่อไหร่) เสมอ

## 3. โมดูลบริหารจัดการ (Specialized Modules)

### 3.1 ระบบบุคลากร (HR Module)
- ใช้สำหรับจัดการข้อมูลอาจารย์และเจ้าหน้าที่ (รหัส, สังกัด, วุฒิการศึกษา)
- รองรับการคำนวณ Generation และปีที่เกษียณอายุโดยอัตโนมัติ

### 3.2 ระบบนิสิต (Student Module)
- จัดการประวัตินิสิตบัณฑิตศึกษา ความก้าวหน้า และผลงานตีพิมพ์
- มีระบบ Autocomplete ช่วยกรอกชื่ออาจารย์ที่ปรึกษาและสาขาวิชา

### 3.3 ระบบงานวิจัย (Research Module)
- เชื่อมต่อ Scopus API เพื่อสืบค้นและดึงผลงานตีพิมพ์ระดับสากล
- รองรับการส่งออกรายงานสรุปผลงานวิจัยในรูปแบบ A4 เพื่อประกอบรายงาน SAR

---

## 4. การจัดการข้อมูลจำนวนมาก (Import / Export / Script)
### 4.1 การนำเข้าข้อมูลด้วย Excel
- ใช้ช่องทางเมนู **Import** ในโมดูลที่เกี่ยวข้อง
- **Template**: แนะนำให้ดาวน์โหลด Template มาตรฐาน (`v1.1b`) เพื่อให้ข้อมูลตรงตามฟิลด์
- ระบบมีการตรวจซ้ำ (Dedup) ตามรหัสนิสิตหรือรหัสบุคลากร

### 4.2 Script แปลงไฟล์ (convert_old_to_template.py)
- สำหรับแอดมิน: ใช้ Python Script ในการแปลงไฟล์ดิบจากกองบริการการศึกษา/บัณฑิตวิทยาลัย ให้เป็นรูปแบบ Import ที่ระบบต้องการได้ทันที

### 4.3 การสำรองข้อมูล (Backup)
- แนะนำให้ส่งออก (Export) ข้อมูลเป็นไฟล์ JSON หรือ Excel อย่างสม่ำเสมอผ่านเมนู **Reports**
