## 1. ภาพรวมโมดูลงานวิจัย (Overview)

โมดูลงานวิจัย (Research Module) ถูกออกแบบมาเพื่ออำนวยความสะดวกในการติดตามผลงานตีพิมพ์ของบุคลากร โดยเน้นการดึงข้อมูลอัตโนมัติจาก Scopus เพื่อความถูกต้องและรวดเร็ว

### ฟังก์ชันหลักในเวอร์ชัน 1.3:
1. **Scopus Intelligence Search**: สืบค้นงานวิจัยแบบเรียลไทม์ตามปี หรือสังกัด
2. **Infinite Pagination**: ดึงข้อมูลได้ครบถ้วนไม่จำกัดจำนวน (ครั้งละ 25 รายการ)
3. **A4 Smart Export**: ส่งออกผลการค้นหาเป็นรายงานรูปแบบ A4 (Google Sans Style) พร้อมพิมพ์ทันที

---

## 2. การสืบค้นข้อมูลจาก Scopus

### 2.1 โหมดการทำงาน
ระบบแบ่งการทำงานเป็น 2 โหมด:
- **แสดงผลเท่านั้น (Search Overlay)**: สำหรับดูข้อมูลและพิมพ์รายงาน โดยไม่บันทึกลงฐานข้อมูล
- **สืบค้นเพื่อนำเข้า (Import Mode)**: สำหรับตรวจสอบความซ้ำซ้อนและเตรียมนำเข้าข้อมูลสู่ Firestore (Phase II)

### 2.2 การกำหนดขอบเขตการค้นหา
- **คำสืบค้น (Query)**: ระบุชื่อผู้แต่ง, ชื่อบทความ หรือ Keywords
- **สังกัด (Scope)**: เลือกค้นหาเฉพาะ "คณะสัตวแพทยศาสตร์" หรือ "มหาวิทยาลัยเกษตรศาสตร์" ทั้งหมด
- **ปีที่พิมพ์ (Year)**: ระบุปีที่ต้องการ หรือเลือก "ทุกปี"

### 2.3 ระบบ Pagination (Load More)
เนื่องจากข้อจำกัดของ Scopus API (Complete View) ระบบจะดึงข้อมูลครั้งละ 25 รายการ:
1. หากพบข้อมูลมากกว่า 25 รายการ ปุ่ม **"โหลดข้อมูลเพิ่ม (+25 รายการ)"** จะปรากฏขึ้น
2. เมื่อดึงข้อมูลครบถ้วน ระบบจะแสดง Badge สีเขียว **"ตรวจสอบแล้ว: ดึงข้อมูลครบถ้วนทั้งหมด"**

## 3. การส่งออกรายงาน (Export A4)

ระบบรองรับการสร้างรายงานสรุปผลการค้นหาเพื่อใช้ประกอบการทำรายงาน EdPEx หรือรายงานประจำปี:
1. กดปุ่ม **"Export (A4)"** บนแถบเครื่องมือ
2. ระบบจะเปิด Tab ใหม่พร้อมจัดหน้ากระดาษแบบ A4
3. **ดีไซน์**: ใช้ฟอนต์ Inter และ Prompt (Google Sans Style) เพื่อความสวยงามและทันสมัย
4. **การพิมพ์**: มีปุ่ม "พิมพ์รายงาน" สำหรับส่งคำสั่งพิมพ์ไปยัง Printer หรือบันทึกเป็น PDF

---

## 4. มาตรฐานและความถูกต้อง (ALCOA+)

- **Attributable**: ข้อมูลที่สืบค้นระบุแหล่งที่มาจาก Scopus API โดยตรง
- **Legible**: จัดรูปแบบรายงานด้วยฟอนต์ Sans-serif ที่อ่านง่ายและแบ่งสัดส่วนชัดเจน
- **Contemporaneous**: บันทึกวันและเวลาที่ส่งออกรายงานกำกับไว้ทุกฉบับ
- **Original**: ข้อมูลดิบ (Raw Data) จาก Scopus ถูกเก็บรักษาไว้ครบถ้วนใน State ของระบบ

---

## 5. ข้อแนะนำทางเทคนิค

- **API Rate Limit**: ระบบมีการจัดการคำสั่งเรียก API อย่างเหมาะสมเพื่อไม่ให้เกินโควตาของคณะ
- **Browsing Safety**: หากเลือกค้นหาแบบ "มหาศาล" (เช่น ทั่วทั้งมหาวิทยาลัย ทุกปี) ระบบจะมีคำเตือนให้ระบุเงื่อนไขเพิ่มเพื่อป้องกันระบบค้าง
